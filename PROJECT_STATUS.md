## 3. Security review (код і залежності)

### Що перевірено
- Перевірка залежностей: `npm audit` -> `0 vulnerabilities`
- Перевірка зовнішніх посилань `target="_blank"`: використовується `rel="noopener noreferrer"`
- Перевірка JS форми:
  - дані форми виводяться через `textContent`
  - для `mailto` використовується URL-encoding
  - є базова валідація обов'язкових полів
- Перевірка фронтенду на явні XSS-точки:
  - небезпечного рендерингу користувацького HTML не виявлено

### Поточний ризик-профіль
- Критичних/високих вразливостей у коді не виявлено
- Залишкові ризики (інфраструктурні, не на рівні шаблонів HTML):
  - відсутній централізований CSP/HSTS/X-Frame-Options/Referrer-Policy на рівні хостингу
  - фінальна безпека залежить від конфігурації прод-сервера/платформи деплою

### Рекомендації перед продом
- На хостингу ввімкнути/додати:
  - `Strict-Transport-Security`
  - `Content-Security-Policy` (мінімум: обмежити `default-src 'self'`, дозволи для maps/video)
  - `X-Content-Type-Options: nosniff`
  - `Referrer-Policy: strict-origin-when-cross-origin`
  - `Permissions-Policy` (мінімально необхідні дозволи)
- Переконатися, що є 301-редірект:
  - `http -> https`
  - `non-www <-> www` (обрати одну канонічну версію)

## 5. Технічний чекліст релізу
- `npm install`
- `npm run build`
- На хостинг завантажувати саме вміст `dist/`, а не корінь репозиторію
- Перевірити в `dist/`:
  - `sitemap.xml`
  - `robots.txt`

- `npm run preview` і ручна перевірка навігації/форми/CTA

## 6. Ключові файли
- `src/pages/*.html` — сторінки сайту
- `src/pages/poslugy/*.html` — деталки категорій послуг
- `src/styles/main.css` — глобальні стилі
- `src/js/main.js` — UI-логіка (меню, reveal, соцблок у футері)
- `src/js/form.js` — логіка форми звернення
- `scripts/generate-sitemap.mjs` — генерація sitemap + flatten
- `vite.config.js` — multi-page build entry points
- `public/robots.txt` — robots
