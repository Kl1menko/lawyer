## F) Decisions & rationale
- Обрано чистий CSS замість Tailwind: менше залежностей, менше JS/CSS-овергеду, контрольований critical path.
- Multi-page HTML через Vite `rollupOptions.input`: SEO-friendly окремі URL без SPA.
- Додано post-build flatten у `scripts/generate-sitemap.mjs`, щоб зберегти розробницьку структуру `src/pages`, але отримати production URL на рівні `dist/*.html`.
- Контактна форма через `mailto:` генерацію в JS: без бекенду, але з можливістю швидкої заміни на Formspree/Netlify у майбутньому.
- Системні шрифти: швидший старт, без зовнішніх font-CDN.
- Поступове SEO-масштабування: спочатку якісні категорії послуг і сильна внутрішня перелінковка, далі — сторінка вартості, підпослуги, блог.
